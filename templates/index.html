Squash me before pull request!

This is a simple tool to allow having GitHub pull requests marked as success or failure based on whether commits have been squashed or not. The basic idea is to squash couple of your previous commits into one. It’s a great way to group changes before sharing them with others. Thanks to that your commit history will be nice and clean.

Want to give it a try?

  • Get commits

    To squash your last three commits first type in

    git rebase -i HEAD~3

    This will open up editor listing last three commits

    pick f392171 Added new feature X

    pick ba9dd9a Added new elements to page design

    pick df71a27 Updated CSS for new elements

  • Squash it

    Then change it to:

    pick f392171 Added new feature X

    squash ba9dd9a Added new elements to page design

    squash df71a27 Updated CSS for new elements

    Now Your commits will be all squashed into one.

  • Enjoy

    And that’s it. Now crack open a bottle of wine and gaze upon your perfectly clean project log.
